# Промпт для генерации чек-листа тестирования

## Цель
Сгенерировать исчерпывающий чек-лист для функционального тестирования формы создания заявки на перевозку (MVP).

## Роль
Ты — опытный инженер по тестированию ПО (QA Engineer) с экспертизой в области логистики и веб-приложений.

## Контекст системы (MVP)
Разрабатывается форма создания заявки на перевозку для сервиса «УЛ Лайт».

## Спецификация полей формы
1.  **Город отправления**: Поле ввода с автокомплитом. Значения подгружаются из базы данных (выпадающий список).
2.  **Город назначения**: Поле ввода с автокомплитом. Значения подгружаются из базы данных (выпадающий список).
3.  **Вес груза**: Числовое поле ввода. Единица измерения — килограммы (кг). **Максимальное значение: 20 000 кг**.
4.  **Объем груза**: Числовое поле ввода. Единица измерения — кубические метры (м³). **Максимальное значение: 82 м³**.
5.  **Дата погрузки**: Поле выбора даты (Datepicker). Доступны для выбора только сегодняшняя дата и будущие даты (прошедшие даты выбрать нельзя).
6.  **Кнопка «Рассчитать стоимость»**: Отправляет POST-запрос к API. В ответе приходит цена в рублях (успех) или сообщение об ошибке (например, если маршрут не найден).

## Требования к чек-листу
Необходимо составить структурированный чек-лист, который включает:

1.  **Позитивные сценарии**: Счастливый путь (Happy Path) и корректная работа всех полей.
2.  **Негативные сценарии**: Проверка валидации полей, ошибок API, некорректного ввода.
3.  **Граничные значения**: Уделить особое внимание значениям: 0, 1, 19999, 20000, 20001 для веса; 0, 1, 81, 82, 83 для объема; а также работе датапикера с датами.

## Инструкция по оформлению
Оформи ответ в виде таблицы со следующими колонками:
| ID | Название проверки | Тип сценария | Предусловия (опционально) | Шаги | Ожидаемый результат |
