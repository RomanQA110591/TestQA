# Негативные проверки и ожидаемое поведение системы

| № | Негативный сценарий | Ожидаемое поведение системы |
|---|---------------------|-------------------------------|
| 1 | Отсутствие обязательного атрибута `ВесГруз` в элементе `ОпГруз` при формировании Титула 1 | XML не проходит XSD-валидацию. Система возвращает ошибку: «Поле "Вес груза" (ВесГруз) обязательно для заполнения». Отправка документа отклоняется. |
| 2 | Указание даты `ДатаПогруз` в недопустимом формате (например, 25/06/2026) или за пределами 1900–2099 | Ошибка валидации формата. Система отображает сообщение: «Дата погрузки должна быть в формате ДД.ММ.ГГГГ (например, 25.06.2026)». |
| 3 | В поле `ИННЮЛ` для грузоотправителя указать 11 цифр (не 10) | XSD-ограничение не пропускает. Ошибка: «ИНН должен содержать 10 цифр (для юридического лица)». |
| 4 | Перевозчик в `ON_TRNVPRPRV` указывает `СодОпер`, отличный от предопределённого значения (любая другая строка) | XSD-ограничение (enum) не принимает значение. Ошибка: «Недопустимое значение для поля "Содержание операции". Допустимо только: "груз принят к перевозке ..."». |
| 5 | При попытке изменить грузополучателя после подписания Титула 1 отправляется `ON_TRNVPRPEREADR`, но в поле `ИдФайлИнфГО` указан неверный идентификатор ранее созданного файла | Система не находит исходный Титул 1. Ошибка: «Не найден исходный документ (ИдФайлИнфГО) для изменения». Изменение не применяется. |
| 6 | Грузополучатель отправляет `ON_TRNVPRGRPO` с `СодОпер=2` (отказ), но не заполняет `ПричОтк` и `ОпПричОтк` (причина отказа) | Schematron-правило нарушено. Ошибка: «При заполнении отказа от приёмки необходимо указать причину (ПричОтк) и описание (ОпПричОтк)». |
| 7 | В поле `НалСт` (налоговая ставка) в `ON_TRNVPRFINSOST` указано значение «15%» (не входит в разрешённый список: 0%, 5%, 7%, 9,09%, 10%, 16,67%, 20%, 5/105, 7/107, 10/110, 20/120, без НДС, НДС исчисляется налоговым агентом) | Ошибка валидации перечисления. Система не принимает файл, сообщает: «Недопустимое значение для налоговой ставки». |
| 8 | Отправка файла с именем, не соответствующим атрибуту `ИдФайл` (например, файл `ABC123.xml`, а `ИдФайл` = `XYZ789`) | Schematron-проверка в XSD: `usch:getFileName() = @ИдФайл`. Ошибка: «Имя файла обмена не совпадает со значением элемента ИдФайл». Документ отвергается. |
| 9 | Подписание Титула 2 с использованием некорректного сертификата (не принадлежит перевозчику, или срок истёк) | Оператор ЭДО возвращает ошибку проверки подписи. Документ не принимается, статус не меняется. |
| 10 | Повторная отправка уже зарегистрированного Титула 1 (дубликат) | Система идентифицирует дубликат по УИД или уникальной комбинации полей. Отклоняет с сообщением: «Документ с таким УИД уже существует». |
| 11 | В поле `КодТовНом` (ТН ВЭД) указать значение длиной 11 символов (максимум 10) | Ошибка длины строки. XSD: `maxLength="10"`. Система уведомляет о превышении максимально допустимой длины. |
| 12 | Для поля `КолМестГр` (количество мест груза) указать дробное число (например, 5.5) при целочисленном типе `xs:integer` | XSD-валидация: тип `xs:integer` не допускает дробную часть. Ошибка преобразования типа. |
