# Тестирование бизнес-процесса электронной транспортной накладной (ЭТрН)

## Ход решения

1. **Анализ требований**  
   - Изучено описание BPMN-диаграммы процесса ЭТрН (4 участника: грузоотправитель, перевозчик, грузополучатель, оператор ЭДО; ключевые этапы: создание Титула 1 → погрузка (Титул 2) → транспортировка → разгрузка (принятие решения) → Титул 3 → регистрация в ГИС ЭПД).  
   - Проанализированы XSD-схемы, предоставленные ФНС (9 схем для разных типов файлов обмена: информация грузоотправителя, перевозчика, изменения грузополучателя/порта, подтверждения, замены судна, информация грузополучателя, выдача груза, финансовое состояние и подтверждение). В фокусе – схема `ON_TRNVPRGO.xsd` (Титул 1) и `ON_TRNVPRPRV.xsd` (Титул 2), а также схемы для последующих этапов.

2. **Подход к тестированию**  
   - Составлен **чек-лист** для покрытия основного happy path, альтернативных сценариев и граничных значений.  
   - Детализированы **тест-кейсы** в формате: ID, предварительные условия, шаги, ожидаемый результат.  
   - На основе XSD выделены **ключевые проверки формы/документа**: обязательность полей, форматы данных (ИНН, КПП, дата/время, числовые значения), ограничения по длине, перечисления (enum).  
   - Описаны **негативные проверки** с ожидаемой реакцией системы (ошибки валидации, отказ в обработке).  
   - Подготовлены два **баг-репорта** с указанием серьёзности и приоритета.  
   - Приведены **промпты к ИИ** для генерации чек-листов и тест-кейсов, а также **архитектурные кейсы** (тестирование статусной модели, интеграции с ГИС ЭПД, подписей и т.д.).

3. **Артефакты**  
   - `checklists.md` – чек-лист функциональных проверок.  
   - `test_cases.md` – детальные тест-кейсы для основного бизнес-процесса.  
   - `field_validation.md` – проверки полей на основе XSD.  
   - `negative_tests.md` – набор негативных сценариев.  
   - `bug_reports.md` – два баг-репорта.  
   - `ai_prompts.md` – промпты и полученные ответы от ИИ.  
   - `architecture_answers.md` – архитектурные кейсы тестирования ЭТрН.
