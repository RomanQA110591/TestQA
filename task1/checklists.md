# Чек-лист тестирования бизнес-процесса ЭТрН

## 1. Создание и отправка Титула 1 (грузоотправитель)
- [ ] Формирование XML-файла в соответствии с XSD `ON_TRNVPRGO`.
- [ ] Заполнение всех обязательных полей (КНД=1110430, НаимТрН, ДатаИнфГО, ВрИнфГО, НомерТрН, ДатаТрН, ДатаПогруз, порты, сведения о грузоотправителе, грузополучателе, перевозчике, сведения о грузе (ОпГруз с СостГруз, СпУпак, ВесГруз и т.д.), сопроводительные документы).
- [ ] Валидация форматов: дата (ДД.ММ.ГГГГ), время (ЧЧ:ММ:СС), ИНН/КПП, числовые поля (вес, объём, плотность).
- [ ] Подписание файла усиленной квалифицированной электронной подписью (УКЭП).
- [ ] Отправка через оператора ЭДО перевозчику и (опционально) иным получателям.
- [ ] Проверка присвоения статуса «Ожидает отгрузки» в ГИС ЭПД.

## 2. Этап погрузки (формирование Титула 2)
- [ ] Прибытие водителя на погрузку (логистика – имитация).
- [ ] Контроль процесса погрузки (внесение фактических данных: объём, вес, количество мест).
- [ ] Формирование XML по схеме `ON_TRNVPRPRV` (информация перевозчика).
- [ ] Заполнение обязательных атрибутов: УИД_ТрН (повторяет УИД из Титула 1), СодОпер="груз принят к перевозке...", СвСудно, ТарифОтм.
- [ ] При наличии оговорок – заполнение `ЗамПрвПрием` (с указанием кодов оговорок 01–12).
- [ ] Подписание УКЭП перевозчика.
- [ ] Отправка грузоотправителю (и иным получателям).
- [ ] Изменение статуса на «В пути» (или «Груз принят перевозчиком» – по логике системы).

## 3. Транспортировка и разгрузка
- [ ] Прибытие на точку разгрузки (логистика).
- [ ] Проверка документов грузополучателем.
- [ ] Осмотр груза, сверка с сопроводительными документами.

## 4. Принятие/непринятие груза грузополучателем
### Happy path (груз принят полностью)
- [ ] Грузополучатель подтверждает приёмку: заполнение `СодИнфГП` (схема `ON_TRNVPRGRPO`), СодОпер=1.
- [ ] Указание фактических дат/времени, массы, количества мест, состояния груза.
- [ ] Подписание и отправка перевозчику.
- [ ] Перевозчик формирует `ON_TRNVPRVGP` (информация о выдаче груза) с СодОпер=1.
- [ ] Статус ЭТрН → «Груз принят грузополучателем».
- [ ] Формирование Титула 3 (фиксация разгрузки) – вероятно, в рамках тех же документов или отдельного файла.
- [ ] Передача всех титулов в ГИС ЭПД через оператора ЭДО.
- [ ] Получение подтверждения регистрации.

### Альтернативные сценарии
- [ ] Отказ в приёмке (СодОпер=2) с указанием причины отказа.
- [ ] Частичная приёмка (СодОпер=3) с указанием принятой массы/количества.
- [ ] Оговорки перевозчика при выдаче (`ЗамПрвВыд` в `ON_TRNVPRVGP`).

## 5. Смена статусов и регистрация в ГИС ЭПД
- [ ] Статус «Ожидает отгрузки» → после подписания Титула 1.
- [ ] Статус «Груз принят перевозчиком» → после получения Титула 2.
- [ ] Статус «В пути» → после отправки Титула 2.
- [ ] Статус «Груз принят грузополучателем» → после успешного завершения.
- [ ] Статус «Отказ в приёмке» → при СодОпер=2.
- [ ] Статус «Частично принят» → при СодОпер=3.
- [ ] Проверка отображения всех реквизитов (УИД, номера, даты, участники, груз) в личном кабинете каждого участника.

## 6. Обработка ошибок
- [ ] Отправка XML, не соответствующего XSD (отсутствие обязательных полей, неверный формат) → ошибка валидации, документ не принимается.
- [ ] Попытка отправить Титул 2 без ссылки на УИД существующего Титула 1 → ошибка «Документ не найден».
- [ ] Подписание документа лицом, не имеющим полномочий (некорректный сертификат) → ошибка подписи.
- [ ] Попытка повторной отправки уже подписанного документа → система должна отклонить (дубль).
- [ ] Истечение времени ожидания подписи → прерывание сессии, уведомление.
- [ ] Сетевые ошибки при передаче через оператора ЭДО → механизм повторных попыток, логирование.

## 7. Граничные значения и дополнительные проверки
- [ ] Минимальная/максимальная длина текстовых полей (например, НаимГруз: 1–255 символов).
- [ ] Максимальные значения числовых полей (ВесГруз до 18 знаков, из них 4 дробных; ОбъемНом до 18 знаков, 6 дробных).
- [ ] Специальные значения: «без номера» для реквизитов документа, «Отсутствует» для маркировки.
- [ ] Коды из классификаторов (ОКПО, ОКВ, ОКСМ, ТН ВЭД, коды оговорок) – проверка допустимых значений.
- [ ] Наличие дополнительных получателей (W=1) – поле `ИдПолИной` обязательно.
