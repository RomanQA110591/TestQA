# 1. GET `/api/v2/orders` – получение списка заказов

## Описание
Возвращает список всех заказов (не удалённых логически).  
Поддерживает стандартные OData‑параметры фильтрации, сортировки и пагинации.

## Тестовые сценарии

| № | Сценарий | Ожидаемый HTTP‑статус | Пояснение |
|---|----------|------------------------|-----------|
| 1.1 | Успешное получение списка заказов | **200 OK** | Возвращается массив заказов в теле ответа |
| 1.2 | Список заказов пуст (нет записей) | **200 OK** | Возвращается пустой массив `[]` |
| 1.3 | Некорректные параметры запроса (например, невалидный `$filter`) | **400 Bad Request** | Ошибка валидации параметров запроса |
| 1.4 | Отсутствует или недействителен токен авторизации | **401 Unauthorized** | Требуется аутентификация |
| 1.5 | Недостаточно прав для просмотра списка заказов | **403 Forbidden** | Токен валиден, но прав недостаточно |
| 1.6 | Внутренняя ошибка сервера / сбой БД | **500 Internal Server Error** | Неожиданная ошибка на стороне сервера |

## Примечания
- При ошибках возвращается JSON с полем `error`.
- Логически удалённые заказы **не** включаются в список.
