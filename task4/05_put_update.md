# 5. PUT `/api/v2/orders/{id:uuid}` – полное обновление заказа

## Описание
Полностью заменяет существующий заказ данными из тела запроса.  
Требует указания всех полей (idempotent).

## Тестовые сценарии

| № | Сценарий | Ожидаемый HTTP‑статус | Пояснение |
|---|----------|------------------------|-----------|
| 5.1 | Успешное обновление существующего заказа | **200 OK** | Возвращается обновлённая read‑model |
| 5.2 | Успешное обновление, read‑model не возвращается | **204 No Content** | Обновление выполнено, тело ответа пусто |
| 5.3 | Заказ с указанным ID не найден | **404 Not Found** | Ресурс не существует |
| 5.4 | Заказ был логически удалён | **410 Gone** | Ресурс удалён, обновление невозможно |
| 5.5 | Некорректный формат UUID в пути | **400 Bad Request** | Ошибка валидации параметра пути |
| 5.6 | Тело запроса не соответствует схеме | **400 Bad Request** | Ошибка валидации тела запроса |
| 5.7 | Нарушение бизнес-правил | **422 Unprocessable Entity** | Семантическая ошибка |
| 5.8 | Отсутствует или недействителен токен авторизации | **401 Unauthorized** | Требуется аутентификация |
| 5.9 | Недостаточно прав для обновления заказа | **403 Forbidden** | Доступ запрещён |
| 5.10 | Внутренняя ошибка сервера / сбой БД | **500 Internal Server Error** | Неожиданная ошибка |

## Примечания
- Использование `PUT` для обновления допускается, но предпочтительнее `PATCH` (согласно Microsoft Graph). В рамках данного проекта принято `PUT`.
- При `204 No Content` тело ответа должно быть пустым.
