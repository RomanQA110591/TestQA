# 8. PUT `/api/v2/orders/{id:uuid}/undelete` – восстановление логически удалённого заказа

## Описание
Восстанавливает ранее удалённый заказ, делая его снова доступным для чтения и обновления.

## Тестовые сценарии

| № | Сценарий | Ожидаемый HTTP‑статус | Пояснение |
|---|----------|------------------------|-----------|
| 8.1 | Успешное восстановление удалённого заказа | **200 OK** | Заказ восстановлен, возвращена обновлённая read‑model |
| 8.2 | Заказ с указанным ID не найден | **404 Not Found** | Ресурс не существует |
| 8.3 | Заказ не был удалён (активен) | **409 Conflict** | Конфликт: заказ уже активен |
| 8.4 | Некорректный формат UUID в пути | **400 Bad Request** | Ошибка валидации |
| 8.5 | Отсутствует или недействителен токен авторизации | **401 Unauthorized** | Требуется аутентификация |
| 8.6 | Недостаточно прав для восстановления заказа | **403 Forbidden** | Доступ запрещён |
| 8.7 | Внутренняя ошибка сервера / сбой БД | **500 Internal Server Error** | Неожиданная ошибка |

## Примечания
- После успешного восстановления `GET /{id}` должен возвращать `200 OK`.
- Повторный вызов `undelete` для уже активного заказа возвращает `409 Conflict`.
