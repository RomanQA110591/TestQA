# 6. DELETE `/api/v2/orders/{id:uuid}` – логическое удаление заказа

## Описание
Помечает заказ как удалённый (логическое удаление).  
Физического удаления из БД не происходит – заказ становится недоступным для чтения и обновления, но может быть восстановлен через `PUT /undelete`.

## Тестовые сценарии

| № | Сценарий | Ожидаемый HTTP‑статус | Пояснение |
|---|----------|------------------------|-----------|
| 6.1 | Успешное логическое удаление заказа | **204 No Content** | Ресурс помечен как удалённый, тело ответа пусто |
| 6.2 | Заказ с указанным ID не найден | **404 Not Found** | Ресурс не существует |
| 6.3 | Заказ уже был логически удалён | **410 Gone** | Ресурс уже удалён |
| 6.4 | Некорректный формат UUID в пути | **400 Bad Request** | Ошибка валидации параметра пути |
| 6.5 | Отсутствует или недействителен токен авторизации | **401 Unauthorized** | Требуется аутентификация |
| 6.6 | Недостаточно прав для удаления заказа | **403 Forbidden** | Доступ запрещён |
| 6.7 | Внутренняя ошибка сервера / сбой БД | **500 Internal Server Error** | Неожиданная ошибка |

## Примечания
- После успешного удаления последующие `GET /{id}` должны возвращать `410 Gone`.
- Повторный `DELETE` для уже удалённого заказа также должен возвращать `410 Gone`.
