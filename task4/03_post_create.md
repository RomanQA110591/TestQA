# 3. POST `/api/v2/orders/` – создание заказа (системный ID)

## Описание
Создаёт новый заказ, присваивая ему системный идентификатор.  
Тело запроса должно соответствовать схеме заказа.

## Тестовые сценарии

| № | Сценарий | Ожидаемый HTTP‑статус | Пояснение |
|---|----------|------------------------|-----------|
| 3.1 | Успешное создание заказа | **201 Created** | Ресурс создан, в ответе может быть ссылка на созданный ресурс |
| 3.2 | Тело запроса не соответствует схеме (отсутствуют обязательные поля, неверный тип данных) | **400 Bad Request** | Ошибка валидации тела запроса |
| 3.3 | Нарушение бизнес-правил (например, отрицательная сумма) | **422 Unprocessable Entity** | Синтаксис верен, но семантика некорректна |
| 3.4 | Отсутствует или недействителен токен авторизации | **401 Unauthorized** | Требуется аутентификация |
| 3.5 | Недостаточно прав для создания заказа | **403 Forbidden** | Доступ запрещён |
| 3.6 | Внутренняя ошибка сервера / сбой БД | **500 Internal Server Error** | Неожиданная ошибка на стороне сервера |

## Примечания
- При успешном создании в ответе рекомендуется возвращать `Location`-заголовок с URL нового ресурса.
- `201` используется, так как есть парный метод `GET /{id}` для последующего получения.
