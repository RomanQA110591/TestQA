# 7. HEAD `/api/v2/orders/{id:uuid}` – проверка существования заказа

## Описание
Позволяет проверить существование заказа без получения его тела.  
Возвращает те же заголовки, что и `GET /{id}`, но без тела.

## Тестовые сценарии

| № | Сценарий | Ожидаемый HTTP‑статус | Пояснение |
|---|----------|------------------------|-----------|
| 7.1 | Заказ существует | **204 No Content** | Заголовки возвращены, тело отсутствует |
| 7.2 | Заказ не найден | **404 Not Found** | Ресурс не существует |
| 7.3 | Заказ логически удалён | **410 Gone** | Ресурс удалён |
| 7.4 | Некорректный формат UUID в пути | **400 Bad Request** | Ошибка валидации |
| 7.5 | Отсутствует или недействителен токен авторизации | **401 Unauthorized** | Требуется аутентификация |
| 7.6 | Недостаточно прав для проверки заказа | **403 Forbidden** | Доступ запрещён |
| 7.7 | Внутренняя ошибка сервера / сбой БД | **500 Internal Server Error** | Неожиданная ошибка |

## Примечания
- `HEAD` должен возвращать те же заголовки (`ETag`, `Last-Modified` и т.д.), что и `GET`.
- Метод полезен для проверки доступности ресурса без загрузки данных.
