# 2. GET `/api/v2/orders/{id:uuid}` – получение заказа по ID

## Описание
Возвращает полную информацию о конкретном заказе по его уникальному идентификатору.

## Тестовые сценарии

| № | Сценарий | Ожидаемый HTTP‑статус | Пояснение |
|---|----------|------------------------|-----------|
| 2.1 | Успешное получение заказа по существующему ID | **200 OK** | Заказ найден и возвращён в теле ответа |
| 2.2 | Заказ с указанным ID не найден | **404 Not Found** | Ресурс не существует |
| 2.3 | Заказ был логически удалён (`DELETE` + `PUT /undelete`) | **410 Gone** | Ресурс был удалён и более недоступен |
| 2.4 | Некорректный формат UUID в пути | **400 Bad Request** | Ошибка валидации параметра пути |
| 2.5 | Отсутствует или недействителен токен авторизации | **401 Unauthorized** | Требуется аутентификация |
| 2.6 | Недостаточно прав для просмотра данного заказа | **403 Forbidden** | Доступ к ресурсу запрещён |
| 2.7 | Внутренняя ошибка сервера / сбой БД | **500 Internal Server Error** | Неожиданная ошибка на стороне сервера |

## Примечания
- Для несуществующего ID возвращается `404`, для удалённого – `410`.
- Ответ с `410` должен содержать пояснение, что ресурс был удалён.
